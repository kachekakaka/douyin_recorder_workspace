# Douyin Recorder v0.1.1

> Release notes draft. No `v0.1.1` tag or GitHub Release exists yet.

v0.1.1 is a stability and operations update for the single-host Windows recorder released as
v0.1.0. It does not change the provisional recipient protocol contract.

## Highlights

- redacted runtime diagnostics with real SQLite schema and migration checks;
- production runtime backup followed by isolated restore verification;
- read-only database maintenance planning;
- explicitly confirmed checkpoint/optimize maintenance with a verified backup;
- database-lock and migration-corruption fault injection;
- repeated recording and postprocess startup recovery with idempotency validation;
- Windows `operations.bat`, `diagnostics.bat` and `maintenance.bat` entrypoints;
- portable-package clean-extraction verification of the new operations commands.

## Windows operations

```bat
operations.bat diagnostics
operations.bat backup
operations.bat maintenance-plan
operations.bat maintenance-apply I_HAVE_STOPPED_THE_APP
```

Maintenance apply mode must only be used after the application is completely stopped. It creates
and verifies a full runtime backup before acquiring an exclusive SQLite lock. It runs only WAL
checkpoint and `PRAGMA optimize`; it does not delete business data, media, or execute VACUUM.

## Compatibility

- existing v0.1.0 SQLite schema remains version 6;
- existing rooms, sessions, recipient intervals, jobs and media indexes are preserved;
- existing v0.1.0 Release and annotated tag remain immutable;
- the application remains loopback-only.

## Protocol limitation

```text
target_method=WebcastGroupLiveGiftRecipientRecommendMessage
live_verified=false
```

Real recipient fields, empty values, duplicate behavior, reconnect sequences, reason values,
server timestamps and latency distributions still require user-authorized, de-identified,
human-reviewed live evidence tracked by Issue #1.
