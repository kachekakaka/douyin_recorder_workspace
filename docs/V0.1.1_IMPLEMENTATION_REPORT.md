### Phase 3 red-light audit

No gate was skipped or weakened.

```text

```

The embedded Python path was invoked inside a nested `for /f` command while the extraction path
contained Chinese characters. `cmd.exe` truncated the executable path. Timestamp generation was
changed to `PowerShell Get-Date`, and a Unicode-path regression test was added.

```text
30018187289  CI failure
```

A static safety assertion intended to reject the destructive Windows `format` command also
matched the safe `Get-Date -Format` parameter. The assertion was narrowed to commands whose
trimmed line starts with `format` or `format.com`; the safety boundary remains enforced.

```text
30018355164  package failure
```

Timestamp generation and diagnostics succeeded, but maintenance-plan correctly rejected a fresh
package because no SQLite database had been initialized. Portable verification now completes the
loopback health startup first, targets its schema-v6 database, and then executes diagnostics and
maintenance-plan. Diagnostics also honors configured runtime paths without running settings sync.

## Security boundaries

All v0.1.1 public reports exclude private configuration values, absolute runtime paths, Cookie,
token, complete signed URLs, raw payload and raw frame. Runtime backups may contain private
configuration and must remain outside Git.

Database maintenance never deletes business rows or media. It does not execute VACUUM. Write
maintenance fails closed unless a verified backup exists and an exclusive database lock can be
acquired.
