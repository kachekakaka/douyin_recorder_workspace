# 9. 管理 API 与纯网页前端

## 9.1 API

| **方法** | **路径**                              | **用途**                                |
|----------|---------------------------------------|-----------------------------------------|
| GET      | /api/status                           | 服务、FFmpeg、SQLite、磁盘、任务摘要    |
| GET/POST | /api/rooms                            | 查询/新增房间                           |
| PATCH    | /api/rooms/{room_key}                 | 修改启用、画质、协议和检查间隔          |
| POST     | /api/rooms/{room_key}/actions/check   | 立即检查一次                            |
| POST     | /api/rooms/{room_key}/actions/enable  | 启用长期监听                            |
| POST     | /api/rooms/{room_key}/actions/disable | 停用监听；不隐式删除历史                |
| POST     | /api/rooms/{room_key}/actions/stop    | 优雅停止当前录制，可显式结束场次        |
| GET      | /api/sessions                         | 分页查询场次                            |
| GET      | /api/sessions/{id}                    | 场次、媒体、缺口、错误和导出摘要        |
| GET      | /api/sessions/{id}/recipient-timeline | 推荐收礼人区间                          |
| POST     | /api/sessions/{id}/exports            | 创建 index/fast-copy/precise/proxy 任务 |
| GET/POST | /api/jobs / {id}/actions/retry        | 任务列表与重试                          |
| HEAD/GET | /api/media/{id}/stream                | Range 播放；只接受数据库媒体 ID         |
| GET      | /api/events                           | SSE 状态流，带 event id                 |

## 9.2 网页页面

| **页面** | **最低功能**                                                 |
|----------|--------------------------------------------------------------|
| 概览     | 正在录制、降级房间、磁盘、任务积压、最近错误                 |
| 直播间   | 新增 URL、启停、画质、协议、检查间隔、立即检查               |
| 实时场次 | 媒体/IM/recipient 独立状态、当前文件、重连次数、当前推荐对象 |
| 录像库   | 按房间、日期、状态筛选；查看完整媒体、缺口和导出             |
| 场次详情 | 播放、推荐时间线、Unknown 原因、分片、日志摘要               |
| 任务中心 | 切片、转封装、代理、校验、失败重试和取消                     |
| 设置     | 目录、阈值、并发、FFmpeg 版本、备份状态                      |
| 诊断     | 未知 method、解码失败、最近 probe、协议/Proto 版本           |

## 9.3 纯网页实现规则

- FastAPI 直接托管 web/index.html 与 web/assets；前后端同源，不开放宽泛 CORS。

- 使用 \<script type="module"\> 拆分 api.js、router.js、sse.js 和 pages/\*.js；运行时不需要 Node。

- 写请求带 CSRF token；会话 Cookie 为 HttpOnly，服务器模式使用 Secure。

- SSE 事件带 id；客户端使用 Last-Event-ID 重连，历史缺失时重新拉取状态快照。

- 原始媒体优先；浏览器不兼容时按需生成 MP4 代理，代理不覆盖原始文件。
