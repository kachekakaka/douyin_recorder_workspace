## 12.1 Git 跟踪与不跟踪边界

| **Git 必须跟踪**                             | **Git 必须忽略**                                     |
|----------------------------------------------|------------------------------------------------------|
| Python/网页源码、测试、文档、迁移            | 实际 .env、config/config.json、Cookie/secret         |
| \*.default 配置模板与递归补全逻辑            | userdata/\*、SQLite db/wal/shm、日志、缓存、临时文件 |
| requirements 锁文件、Dockerfile、compose     | records/\*、导出、代理、媒体、备份包                 |
| start/update/verify/backup 与内部脚本        | 解压后的 .runtime、.venv、FFmpeg 可执行目录          |
| Proto 生成结果、schema hash、replay fixtures | 真实抓包中不必要的个人信息或完整凭据                 |

实际配置不存在时从 .default 创建；模板增加字段时只补缺失项，保留用户已有值。JSON 使用原子写入和备份，损坏配置不得被默认模板静默覆盖。

## 12.2 根目录用户入口

| **文件**   | **职责**                                                           |
|------------|--------------------------------------------------------------------|
| start.bat  | 准备便携 Python/FFmpeg、同步配置、启动服务并打开网页               |
| update.bat | 拒绝脏工作区；git pull --ff-only；随后执行 verify.bat              |
| verify.bat | 源码结构、依赖、compileall、Ruff、pytest、FFmpeg、前端 JS 语法冒烟 |
| backup.bat | 创建 Git Bundle 与运行数据备份，生成 SHA256SUMS；不把备份写入仓库  |
