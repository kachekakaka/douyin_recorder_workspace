# 17. 风险与应对

| **风险**                | **影响**                   | **应对**                                                          |
|-------------------------|----------------------------|-------------------------------------------------------------------|
| 目标房间不下发推荐消息  | 无法确定对象               | 正常录制，状态保持 Waiting/Unknown；不启用其他推断                |
| 首次连接/重连不重放对象 | 开头或断线后 Unknown       | 接受严格语义；等待下一条变化事件                                  |
| 抖音协议、签名变化      | 状态/IM/解码失败           | Proto 固定版本、未知 method 指标、raw frame fixture、独立更新发布 |
| IM 与画面延迟不同       | 切片边界偏差               | receive_time + 固定 offset；保留原始媒体与实际 PTS                |
| 流地址过期/编码变化     | 媒体断流或无法 copy concat | 重新解析、画质回退、continuity_group、必要时转码                  |
| SQLite 写锁/损坏        | 状态写入失败               | 单 DbWriter、WAL、本地盘、备份 API、JSONL 灾备                    |
| 磁盘爆满                | 全部录制失败               | 软硬阈值、停止新任务、媒体保留策略、外部监控                      |
| GitHub 误删/账号问题    | 源码历史不可用             | 每周 Git Bundle + 源码 ZIP + NAS/云盘；tag 和 SHA 记录            |
| 只备份代码不备份数据    | 房间/场次/时间线/录像丢失  | 代码和运行数据双轨备份，季度恢复演练                              |
| 运行包不可复现          | Windows 无法启动           | 锁定依赖、runtime manifest、SHA、CI 构建和离线恢复测试            |
