# 7. 媒体录制、时间轴与切片

## 7.1 流地址与录制参数

| **参数**                 | **默认值** | **说明**                             |
|--------------------------|------------|--------------------------------------|
| protocol                 | flv        | 低延迟；HLS 为回退                   |
| quality                  | origin     | 不可用时按明确顺序回退               |
| container                | mkv        | 中断恢复较好；原始归档优先           |
| segment_seconds          | 600        | 固定轮转，不随 recipient 切换        |
| copy_codec               | true       | 在线阶段不转码                       |
| re_resolve_on_disconnect | true       | 媒体断线后重新获取流地址             |
| recipient_cut_offset_ms  | 0          | 仅允许人工统一校准，不自动从画面学习 |

## 7.2 FFmpeg Supervisor

| ffmpeg -hide_banner -nostdin -headers "Referer: https://live.douyin.com/\r\nUser-Agent: ...\r\nCookie: ...\r\n" -i "\<stream_url\>" -map 0 -c copy -f segment -segment_time 600 -reset_timestamps 1 -segment_list "segments.csv" -segment_list_type csv -progress pipe:1 "records/\<room\>/\<session\>/media/%05d.mkv" |
|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|

- Python 使用 asyncio.create_subprocess_exec(\*argv)，禁止 shell=True 和字符串拼接。

- stdout 和 stderr 必须同时消费；Cookie、签名和完整流 URL 在日志中脱敏。

- Linux/Windows 均使用独立进程组；停止顺序为优雅退出 → terminate → kill。

- 文件关闭后执行 ffprobe，writing 文件不得被播放、切片或清理。

- 服务重启时扫描 writing 文件：可探测则标记 recovered/truncated；不可探测则移入 quarantine。

## 7.3 时间轴对齐

每个事件同时记录 UTC receive_time 与 monotonic_ns；每个媒体文件记录 requested_start、actual_start、actual_end、首末 progress、first_pts/last_pts、codec_signature 和 continuity_group。切片先按事件时间定位媒体文件，再用 ffprobe/segment CSV 校准实际边界。

| **字段**                              | **作用**                                                      |
|---------------------------------------|---------------------------------------------------------------|
| continuity_group                      | 媒体重连或编码参数变化后分组；不同组不能无条件 concat -c copy |
| requested_start_at_ms                 | Python 启动/切换预期时间                                      |
| actual_start_at_ms / actual_end_at_ms | 基于 segment list 与 ffprobe 的实际媒体范围                   |
| first_pts / last_pts / time_base      | 媒体内部时间轴审计                                            |
| codec_signature                       | 容器、视频/音频编码、分辨率、time_base 摘要                   |

## 7.4 导出模式

| **模式**       | **处理**                             | **使用场景**                               |
|----------------|--------------------------------------|--------------------------------------------|
| index-only     | 只生成时间线和播放器定位，不物理切片 | 最省空间；默认始终可用                     |
| fast-copy      | 按关键帧附近 -c copy                 | 默认导出；速度快、资源低，边界可能有小偏差 |
| precise        | 重编码全部或边缘 GOP                 | 用户显式请求；边界更精确、资源高           |
| playback-proxy | 按需生成 H.264/AAC MP4               | 原始 MKV/编码浏览器不兼容时播放            |
