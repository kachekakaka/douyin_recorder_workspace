# 6. 推荐收礼人状态机

## 6.1 唯一消息与字段

| **字段**               | **用途**          | **存储**                               |
|------------------------|-------------------|----------------------------------------|
| common.msg_id          | 首选去重依据      | TEXT，可为空；不得再定义为 NOT NULL    |
| common.create_time     | 服务器事件时间    | INTEGER，仅审计和延迟观察              |
| recipient_user_id      | 优先唯一标识      | TEXT；有效且不为 "0" 时生成 uid:\<id\> |
| recipient_user_open_id | 备用唯一标识      | user_id 不可用时生成 openid:\<id\>     |
| change_reason_enum     | 原始变化原因      | INTEGER 原样保存，不解释为业务条件     |
| extra                  | 扩展字段          | JSON 文本原样保存                      |
| received_at_ms         | 本机 UTC 接收时间 | 状态生效和默认切片边界                 |
| received_monotonic_ns  | 本机单调时钟      | 当前进程内排序和时钟跳变保护           |
| raw_payload_json       | 完整解码结果      | JSON 文本；另存 payload_hash           |

## 6.3 状态转换

| **当前状态**   | **输入**     | **事务动作**                                | **新状态** |
|----------------|--------------|---------------------------------------------|------------|
| offline        | 检测到开播   | 创建 session；开启 waiting                  | waiting    |
| waiting        | 首条有效 B   | 关闭 waiting；开启 Active(B)                | active(B)  |
| active(A)      | 重复 A       | duplicate_count + 1；不新建区间             | active(A)  |
| active(A)      | 有效 B       | A.end=B.received_at；开启 B                 | active(B)  |
| active(A)      | 空 recipient | 关闭 A；开启 unknown(empty_recipient)       | unknown    |
| waiting/active | IM 断线      | 关闭当前区间；开启 unknown(im_disconnected) | unknown    |
| unknown        | 重连后有效 B | 关闭 Unknown；开启 B                        | active(B)  |
| 任意在线       | 确认下播     | 关闭当前区间和 session                      | offline    |

## 6.4 去重、重复与迟到

- dedup_key = "msg:" + msg_id；msg_id 缺失时使用 "hash:" + payload_hash + 时间桶。

- recipient_events 只保存去重后的 canonical event；重复帧更新 duplicate_count 和 last_received_at_ms，不再次改变状态。

- UNIQUE(session_id, dedup_key) 保证去重，但不会与 duplicate_count 冲突。

- received_at_ms 早于当前区间 started_at_ms 的事件标记 is_late=1，保存但不回滚状态。

- 事件写入与区间闭合在同一 SQLite 事务中，由每房间串行事件队列和全局 DbWriter 保证顺序。
