RoomManager 只做调度与汇总，不直接执行房间业务。某个房间的解析失败、IM 断线、FFmpeg 退出、文件写入错误或后处理失败，都不得取消其他房间任务。

## 4.2 状态拆分

| **状态轴**      | **枚举**                                                      | **说明**          |
|-----------------|---------------------------------------------------------------|-------------------|
| lifecycle_state | disabled / offline / starting / recording / stopping / error  | 房间整体生命周期  |
| media_state     | idle / starting / recording / reconnecting / failed           | FFmpeg 和文件写入 |
| im_state        | disconnected / connecting / connected / reconnecting / failed | 抖音消息连接      |
| recipient_state | waiting / active / unknown                                    | 推荐收礼人时间线  |

前端的 health=healthy/degraded/error 由上述状态派生，不把“RecordingImDisconnected”等组合状态写成唯一事实，避免状态枚举迅速膨胀。

## 4.3 Cookie、ttwid 与并发

- 每个 RoomWorker 有独立游客 Cookie Jar 和 ttwid；同一房间重连可复用，跨房间不共享。

- 登录 Cookie 如需配置，仅保存在受保护的 secrets 文件或环境变量中；状态 API 只返回是否存在与更新时间。

- 状态检查使用 poll_interval ± jitter；启动过程和状态请求分别使用 Semaphore 限流。

- max_active_recordings=0 明确定义为“不额外限制”；大于 0 时达到上限的新开播房间进入 pending_capacity。

- 后处理并发默认 1；在线录制期间禁止高并发精确转码抢占磁盘和 CPU。
