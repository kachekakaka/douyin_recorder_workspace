# 文档说明

本方案仍将“推荐收礼人”定义为业务层唯一身份信号。系统内部统一使用 active_recipient / recipient，避免把协议语义误写成严格意义上的“当前说话人”或“画面 C 位”。界面如显示“当前主播”，必须同时注明来源为“抖音当前推荐收礼人”。
