# 2. 最终技术决策

| **层**      | **采用方案**                        | **说明**                                               |
|-------------|-------------------------------------|--------------------------------------------------------|
| 语言/运行时 | Python 3.12 或 3.13                 | 控制面、网络、状态机和 API 统一 Python；发布锁定版本   |
| Web 后端    | FastAPI + Uvicorn                   | 同源托管 API、SSE、媒体 Range 和静态网页               |
| 异步模型    | 标准 asyncio                        | RoomWorker、IM、FFmpeg 监督和任务调度均使用异步任务    |
| HTTP        | httpx.AsyncClient                   | 状态查询、直播页与流地址解析；统一超时和 Cookie Jar    |
| WebSocket   | websockets                          | 二进制帧、心跳、ACK、断线重连                          |
| Proto       | protobuf 生成 Python 文件           | 生成结果和 schema hash 提交仓库；生产启动时不自动生成  |
| 数据库      | SQLite WAL + aiosqlite              | 单机部署最轻；一条有序写队列；数据库必须位于本地状态盘 |
| 媒体        | FFmpeg / ffprobe                    | 每个在线房间一个进程；在线 stream copy；关闭后校验     |
| 实时推送    | SSE                                 | 单向状态更新足够；断线可重连；前端实现简单             |
| 前端        | HTML + CSS + 原生 JS ES Modules     | 无打包、无 Node 运行依赖；浏览器直接访问               |
| 后处理      | SQLite 持久化 jobs + JobRunner      | 无需 Celery/Redis；重启后可恢复排队                    |
| 部署        | Windows 便携运行包 + Docker Compose | 既能双击运行，也能长期部署在 NAS/服务器                |

## 2.1 明确约束

- Uvicorn 首版固定一个 worker。多个 worker 会重复创建 RoomManager 和 FFmpeg，禁止开启。

- SQLite 只放在本机磁盘的 userdata/；records/ 可以映射 NAS，但不建议把 SQLite 放在 SMB/NFS 上。

- 所有平台 64 位 ID 以字符串传输和存储，前端禁止转为 JavaScript Number。

- Proto 更新是开发发布动作，不是生产运行时动作；每次更新必须通过历史帧 fixture 回放。
