# 16. 验收标准

| **验收项**   | **通过条件**                                                      |
|--------------|-------------------------------------------------------------------|
| 多个房间监听 | 网页可配置多个房间；状态检查带抖动；独立启停                      |
| 并发收流     | 每个在线房间独立 FFmpeg 和 IM；任一异常不终止其他房间             |
| 唯一信号     | 代码和配置不存在 OCR、礼物/弹幕/画面推断 fallback                 |
| 连续媒体     | A→B 切换期间 FFmpeg PID 和媒体连接不变化                          |
| 严格 Unknown | 首事件前、空对象、IM 断线不延续或猜测 recipient                   |
| 去重与乱序   | 重复帧只增加计数；迟到事件不回滚状态                              |
| 恢复         | 主服务重启后不重复录制；writing 文件和 jobs 按规则恢复            |
| 网页         | 手机/桌面可管理房间、查看状态、播放、看时间线和任务               |
| 后处理       | index/fast-copy 可用；precise 按需；原始媒体始终保留              |
| 审计         | 任一 export 可追溯到 interval、event 和 media_files               |
| 资源保护     | 磁盘阈值和并发上限按预设策略降级并告警                            |
| 仓库防丢     | CI 通过；tag、Git Bundle、SHA256 和数据备份齐全；完成一次恢复演练 |

## 16.1 稳定性测试

- 覆盖 IM 反复断连、媒体 404/超时、状态接口失败、SQLite busy、磁盘软硬阈值和主进程重启。

- 覆盖 FFmpeg 异常退出、writing 文件损坏、流地址/清晰度变化和 continuity_group 改变。

- 覆盖 update.bat 脏工作区拒绝、CI 失败、代码 tag 回滚、SQLite 恢复和 Git Bundle 克隆。

- 输出目标硬件的安全同时录制路数、总码率、CPU、内存、磁盘吞吐和文件句柄结论。
