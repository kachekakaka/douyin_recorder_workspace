# 5. 端到端业务流程

> **1.** 服务启动，执行配置同步、数据库迁移、FFmpeg/ffprobe 自检和崩溃恢复；随后创建 RoomManager。
>
> **2.** RoomManager 从 SQLite 加载启用房间，为每个房间创建 RoomWorker。
>
> **3.** Offline Worker 按 poll_interval ± jitter 查询直播状态；失败只影响本房间。
>
> **4.** 检测到开播后，以数据库唯一约束创建一个 active session，获取 room_id/web_rid、标题、封面和流候选。
>
> **5.** 几乎同时启动 FFmpeg 连续录制和抖音 IM WebSocket；recipient 先进入 waiting。
>
> **6.** 收到 WebcastGroupLiveGiftRecipientRecommendMessage 后生成 dedup_key，写入 canonical event，再串行驱动状态机。
>
> **7.** 对象 A→B 时闭合 A 区间并开启 B；FFmpeg 不重启、不换文件，除非固定 segment 轮转。
>
> **8.** IM 断线时立即闭合确定区间并进入 unknown(im_disconnected)；媒体继续录制。
>
> **9.** 媒体断线时关闭 writing 文件、记录 media_gap，重新解析流地址并启动新媒体文件；recipient 状态机独立运行。
>
> **10.** 连续确认下播后关闭当前 interval、session 和媒体，创建校验/索引/fast-copy 任务。
>
> **11.** 网页展示完整录像、时间线、Unknown 段、媒体缺口和导出状态。

## 5.1 下播确认

单次状态接口返回离线不得立即结束场次。默认要求连续 3 次离线，或“媒体流自然结束 + 状态接口确认离线”。若状态接口失败但 FFmpeg 或 IM 仍活跃，应维持场次并告警。
